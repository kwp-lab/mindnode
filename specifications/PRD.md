# 产品核心价值建议书 (Product PRD & Landing Page Strategy)

## 1. 核心定义 (The Identity)

* **产品名称：** MindNode AI (建议：*ThinkFlow*, *NodeMind*, 或 *TreeGPT*)
* **Slogan：** **思维不再线性，灵感不再淹没。** (Stop Chatting, Start Mapping.)
* **Subtitle：** 首款“画布式”AI 思考引擎。将碎片化的对话转变为结构化的知识资产，让 AI 真正深度参与你的逻辑构建。

---

## 2. 核心卖点 (Key Selling Points - KSPs)

### KSP 1: 告别“黑盒”对话，拥抱空间记忆 (Spatial Thinking)

* **痛点：** 传统 AI 对话像瀑布，有用的信息很快被淹没在上下滚动中。
* **方案：** 在无限画布上展开对话。每一个观点都是一个节点，每一条逻辑都有迹可循。
* **价值：** 利用人类的“空间记忆”，通过位置而非时间来管理信息，认知负担降低 60%。

### KSP 2: 原子级精准追问 (Precision Branching)

* **痛点：** AI 回复太长，想针对其中一个词深入探讨，却得写一大段话来解释背景。
* **方案：** **“选中即追问”**。鼠标划选回复中的任何词汇或句子，直接 `Tab` 键开启新分支。
* **价值：** 消除指代不明，让 AI 的注意力（Attention）精准聚焦在你关心的细节上。

### KSP 3: 上下文的“基因继承” (Logical Inheritance)

* **痛点：** 对话越久，AI 越容易“跑题”或产生幻觉。
* **方案：** 每一个子节点天然继承其父节点及路径上的所有背景，自动屏蔽无关分支的噪音。
* **价值：** 保证了深度思考时的逻辑严密性，让 AI 成为你真正的智囊而非复读机。

### KSP 4: 多 Agent 协作，专业角色分工 (Multi-Agent Collaboration)

* **痛点：** 单一 AI 无法同时兼顾多个专业视角，用户需要在不同工具间切换。
* **方案：** 在同一画布中配置多个人格化 Agent（产品经理、设计师、营销专员等），每个 Agent 拥有独立的系统提示词和模型配置。
* **价值：** 一个主题，多个专家视角。让不同领域的 AI 在同一空间协同思考，碰撞出更全面的解决方案。

### KSP 5: 模型自由，打破平台锁定 (Model Provider Freedom)

* **痛点：** 被单一 AI 提供商锁定，无法根据任务特性选择最优模型。
* **方案：** 支持用户自主配置任意云厂商的 API Key（OpenAI、Claude、Gemini、国产大模型等），自由选择模型。
* **价值：** 成本可控、性能可选。让用户真正拥有 AI 的选择权，不同 Agent 可使用不同模型，实现最优性价比组合。

## 3. 功能展示区 (Feature Showcase)

### Showcase 1: 极简交互，丝滑流转 (The Flow)

* **文案：** **“像用导图一样简单，像聊微信一样直接。”**
* **描述：** 点击 `Enter` 整理你的思绪，点击 `Tab` 召唤 AI 的智慧。无需复杂的 Prompt，按键之间，逻辑自成。

### Showcase 2: 划词触发：深度钻取 (The Deep Dive)

* **文案：** **“别只看表面，深入每一个细节。”**
* **视觉描述：** 展示一个动图——用户在 AI 回复中选中“技术选型”四个字，侧边立即生长出一个新的逻辑分支，AI 开始详细拆解技术栈。

### Showcase 3: 工作站：从对话到资产 (The Workspace)

* **文案：** **“你的 AI 思考档案库。”**
* **描述：** 所有的对话不再是临时记录。通过 Workspace 组织你的项目，支持一键导出结构化 Markdown，无缝衔接你的工作流。


### Showcase 4: 多 Agent 协作画布 (The Team Canvas)

* **文案：** **"一个画布，多个大脑。"**
* **视觉描述：** 展示一个产品策划画布，左侧分支由"产品经理 Agent"（GPT-4）分析需求，右侧由"设计师 Agent"（Claude）提供 UI 建议，底部"营销专员 Agent"（国产模型）输出推广方案。不同 Agent 用不同颜色标识。
* **价值：** 像组建虚拟团队一样使用 AI，每个角色各司其职，协同完成复杂项目。

### Showcase 5: 模型自由配置 (The Model Marketplace)

* **文案：** **"你的 AI，你做主。"**
* **描述：** 在设置中添加任意云厂商的 API Key，为不同 Agent 分配不同模型。需要创意？用 Claude。要求速度？选 GPT-3.5。成本敏感？试试国产大模型。一切由你决定。

## 4. 应用场景 (Use Cases - 谁在用它？)

| 场景 | 如何使用 |
| --- | --- |
| **SaaS 创业者** | 梳理产品功能矩阵，针对每个用户路径进行 AI 压力测试。 |
| **科研/学者** | 拆解复杂的论文综述，针对特定的理论点进行划线延伸讨论。 |
| **内容创作者** | 从一个选题出发，通过导图发散出大纲、金句、案例，直接生成文章。 |
| **程序员** | 进行系统架构设计，圈选特定的模块让 AI 提供多种代码实现方案。 |
| **跨界知识攻克** | 快速消化金融、法律、医学等高门槛知识。圈选专业术语 $\rightarrow$ 弹出“降维解释”。 |
| **语言沉浸学习** | 圈选句子中的词组 $\rightarrow$ 分支：用法辨析、地道例句、文化起源。 |
| **产品全流程设计** | 在同一画布中召唤产品经理 Agent 定义需求、设计师 Agent 规划界面、技术 Agent 评估可行性，三方对话形成完整方案。 |
| **多模型对比验证** | 针对同一问题，让不同模型的 Agent 分别给出答案，通过分支对比找到最优解。 |

## 5. 为什么是现在？ (Why Now?)

1. **从 Prompt 工程到交互工程：** 用户不再满足于写长长的指令，他们需要更自然、更精准的交互方式。
2. **从“流”到“存”：** 生成式 AI 正在进入生产力深水区，信息的**结构化**是将其转化为**知识资产**的唯一路径。
